# Программный API обработок КД 3.1 (что вызывается headless)

Выверено по коду EDT-проекта `КД_31_Демопремьер_77УНФ`. Форменные обработки КД
делают тяжёлую работу НЕ в форме, а в экспортных методах ManagerModule/ObjectModule
(форма лишь собирает входные данные и запускает фон). Поэтому ключевые шаги
вызываемы через `:6011` execute_code, минуя UI. file:line - для чтения исходника
при необходимости.

Общий приём headless-вызова форменной обработки:
1. `get_module_structure` модуля формы -> найти серверный метод кнопки
   (`Выполнить*Сервер`).
2. Прочитать его -> увидеть реальный вызов (Export-метод Manager/Object или
   `ДлительныеОперации.ВыполнитьВФоне("Обработки.X.Метод", Параметры)`).
3. Собрать параметры и вызвать метод. Тяжёлые операции (большая конфигурация) -
   через фон с опросом, не синхронно (иначе таймаут toolkit).

## 1. ЗагрузкаСтруктурыКонфигурацииИзФайловXML - загрузка структуры (-> Релиз/Объекты/Свойства)

Заполняет теневую модель (Релиз + Объекты + Свойства + Значения) из исходников
конфигурации. Поддерживает форматы XML-дампа Конфигуратора И EDT.

Экспортный метод (вызывается фоном из формы):
- `ВыполнитьЗагрузкуМетаданных(ОбщиеПеременные, АдресВременногоХранилища)` Export -
  `DataProcessors/ЗагрузкаСтруктурыКонфигурацииИзФайловXML/ManagerModule.bsl:7`.
  `ОбщиеПеременные` = СтруктураПараметров (см. ниже); `АдресВременногоХранилища` -
  адрес для результата.

Структура параметров (форма, `Forms/Форма/Module.bsl:344` `ВыполнитьЗагрузкуСервер`):
- `СпособЗагрузки` - вариант (XML-дамп / EDT)
- `ПомещенныеФайлы` - Массив структур `{ПолноеИмя, Хранение, ЭтоРасширение}`, где
  `Хранение` - **путь к файлу на сервере** (форма пишет файлы во временный каталог
  и подставляет путь)
- `ИмяНовойКонфигурации`, `Конфигурация`, `ИсточникДанных`, `ВариантЗагрузкиДвижений`
- `ТолькоОбновитьПланыОбмена`, `ТолькоДобавлятьНовые` - Boolean
- `ЕстьРасширения` - Boolean; `ИмяКаталогаЗагрузки` - String;
  `КаталогиРасширений` - Массив (если есть расширения)

Ридеры форматов внутри менеджера:
- EDT: `ПрочитатьСведенияОбОбъектеИзEDT` (`ManagerModule.bsl:1142`),
  `ПрочитатьСведенияОСвойствахИзEDT` (`:1616`)
- XML-дамп: `ПрочитатьСведенияОбОбъектеИзXML` (`:934`),
  `ПрочитатьСведенияОСвойствахИзXML` (`:1314`)
- Хелперы поиска файлов: `НайтиПомещенныеФайлыПоИмени` (`:3062`),
  `НайтиПомещенныеФайлыПоМаске` (`:3074`)

Headless-сценарий: собрать `ПомещенныеФайлы` из EDT-исходников УНФ
(`...\Демо_УНФ_3\src`) на сервере КД, вызвать `ВыполнитьЗагрузкуМетаданных`
как фоновое задание (`ДлительныеОперации.ВыполнитьВФоне`) с опросом. УНФ большая -
синхронно нельзя.

#### Проверенный рецепт (выполнено 2026-05-29 на боевой ИБ КД, УНФ ~17к .mdo)

1. **robocopy** только загружаемых видов .mdo во ВРЕМЕННЫЙ каталог (база + расширение
   отдельно). Обязательно во временный: загрузчик в конце делает `УдалитьФайлы(Хранение)`
   по каждому файлу - если `Хранение` указывает на EDT-оригиналы, он их СНЕСЁТ.
   Загружаемые виды: `Catalogs,Documents,Enums,Tasks,BusinessProcesses,ChartsOfAccounts,
   ChartsOfCalculationTypes,ChartsOfCharacteristicTypes,AccountingRegisters,
   AccumulationRegisters,CalculationRegisters,InformationRegisters,ExchangePlans,
   EventSubscriptions,Constants,DefinedTypes,CommonAttributes,Configuration`
   (CommonModules/Roles/Forms и пр. не нужны - загрузчик их не читает).
2. На сервере КД собрать `ПомещенныеФайлы` - массив структур
   `{ПолноеИмя, Хранение, ЭтоРасширение}`, где и `ПолноеИмя`, и `Хранение` = путь к
   temp-копии (матчинг идёт по подстроке вида в `ПолноеИмя`). База - `ЭтоРасширение=Ложь`,
   расширение - `Истина`, через `НайтиФайлы(temp\src\<Вид>, "*.mdo", Истина)`.
3. `СтруктураПараметров`: `СпособЗагрузки=0` (новый Релиз), `ИсточникДанных=1` (EDT),
   `ВариантЗагрузкиДвижений=1`, `ТолькоОбновитьПланыОбмена=Ложь`, `ТолькоДобавлятьНовые=Ложь`,
   `ЕстьРасширения=Истина`, `ИмяКаталогаЗагрузки`=temp-корень базы,
   `КаталогиРасширений`=[temp-корень расширения], `ИмяНовойКонфигурации`=имя Релиза,
   `Конфигурация`=`Справочники.Релизы.ПустаяСсылка()`.
4. Запуск: `ДлительныеОперации.ВыполнитьВФоне("Обработки.ЗагрузкаСтруктурыКонфигурацииИзФайловXML.ВыполнитьЗагрузкуМетаданных", СП, ПарамФон)`.
   В ФАЙЛОВОЙ базе работает по-настоящему АСИНХРОННО (возвращает Статус=Выполняется + JobID).
5. **Опрос завершения - НЕ по счётчикам!** Объекты/Свойства/Значения стабилизируются
   ЗАДОЛГО до конца (после создания объектов идёт длинная фаза заполнения типов 69к
   свойств - счётчики не растут, но задание работает). Признак конца:
   `ФоновыеЗадания.НайтиПоУникальномуИдентификатору(JobID).Состояние <> Активно`, либо
   `РегистрСведений.СостоянияЗагрузокМетаданных` (Измерение `Релиз`, ресурс `Результат`)
   = `Выполнено`. Проверять ошибку через `Задание.ИнформацияОбОшибке`.
6. После загрузки - привязать Релиз к Конвертации:
   `Об=Конв.ПолучитьОбъект(); Об.Конфигурация=Релиз; Об.КонфигурацияКорреспондент=Релиз; Об.Записать();`.

Грабли BSL toolkit-компилятора (фрагмент через Выполнить): `Новый Тип(...).Метод()` -
chaining прямо на конструкторе ЗАПРЕЩЁН («Ожидается Тогда») - присваивать в переменную;
имя переменной не может быть `И` (зарезервировано - «Ожидается имя переменной»);
chaining на результате метода (`Зап.Выполнить().Выгрузить()[0]`) - разрешён.

Результат на УНФ: ~31 мин, Релиз + Объекты=4060, Свойства=69015, Значения=13932,
Демо-свойств=202 (расширение подхватилось).

## 2. ВыгрузкаМодуля - генерация модуля менеджера обмена (-> BSL)

Главный артефакт КД 3.1: текст модуля менеджера обмена для движка `ОбменДаннымиXDTO`.

Экспортные методы:
- `ВыполнитьВыгрузкуМодулей()` Export - `DataProcessors/ВыгрузкаМодуля/ObjectModule.bsl:9`.
  **Основной генератор.** Без параметров - читает реквизиты объекта обработки
  (Конвертация + настройки), возвращает результат (тексты модулей).
- `СформироватьТекстПравила(Параметры)` Export - `ObjectModule.bsl:199`. Генерация
  текста одного правила.
- `ПодготовитьНастройкиВыгрузкиМодулей(Конвертация, ИменаФайловДляСохранения, НастройкиВыгрузки, ОбщиеПеременные = Неопределено)`
  Export - `ManagerModule.bsl:13`. Заполняет `НастройкиВыгрузки` (ИмяКаталога,
  СпособВыгрузки, ВыгрузкаВТХТ/ВыгрузкаВХМЛ, ТегиДляМодуляМенеджера) и
  `ИменаФайловДляСохранения` по реквизитам Конвертации (`ИмяФайлаДляСохранения`,
  `ИспользоватьРазделениеМодуля`, `СпособВыгрузкиРазделенногоМодуля`).
- `ПолучитьТекстЗаголовкаКомандыВыгрузкиМодуля(Конвертация)` Export - `ManagerModule.bsl:64`.

Рецепт (форма, `Forms/Форма/Module.bsl:53` `ВыгрузитьМодулиСервер`):
```bsl
НастройкиВыгрузки = Новый Структура; ИменаФайлов = Новый Структура;
Обработки.ВыгрузкаМодуля.ПодготовитьНастройкиВыгрузкиМодулей(Конв, ИменаФайлов, НастройкиВыгрузки);
Обр = Обработки.ВыгрузкаМодуля.Создать();
Обр.Конвертация = Конв;   // + перенести НастройкиВыгрузки/ИменаФайлов в реквизиты объекта обработки
Результат = Обр.ВыполнитьВыгрузкуМодулей();
```
Для headless нужны точные имена реквизитов объекта обработки `ВыгрузкаМодуля`
(получить через `get_metadata_details Обработки.ВыгрузкаМодуля` - настройки кладутся
в объект перед вызовом). Зависимость: движок `КонвертацияДанныхXDTOСервер`
(`.ТегиДляМодуляМенеджераОбъекта()`).

`СпособВыгрузкиРазделенногоМодуля` (EnumRef.СпособыВыгрузкиРазделенногоМодуля):
`ФайлыTXT`, `ФайлыXML`. Реквизит Конвертации `ИспользоватьРазделениеМодуля`
включает раздельную выгрузку модулей менеджеров.

## 3. Прочие обработки (API не прослежен до конца - пробовать перед использованием)

| Обработка | Назначение | Где искать API |
|---|---|---|
| `ВыгрузкаКонвертацииXML` / `ЗагрузкаКонвертацииXML` | перенос правил между ИБ КД в XML | ManagerModule/ObjectModule |
| `СохранитьКомплектПравилXML` | сохранить комплект правил | -"- |
| `ЗагрузкаМодуляМенеджера` | загрузка готового модуля (конвертация XDTO) | -"- |
| `НастройкаПравилОбмена` / `НастройкаПравилКонвертацииСвойств` | редакторы правил | преимущественно UI (форма) |
| `ВыгрузкаСтруктурыФормата` / `ЗагрузкаСтруктурыФормата` | структура формата ED | ManagerModule/ObjectModule |
| `ПроверкаКонвертации` | валидация правил | -"- |

Перед программным вызовом: `get_module_structure` ManagerModule и ObjectModule,
найти Export-метод, прочитать сигнатуру (как сделано выше для загрузчика и генератора).

## Движок конвертации (общие модули - потребители генерируемого менеджера)

Генерируемый модуль менеджера обмена работает поверх движка `ОбменДаннымиXDTO` и
этих общих модулей:
- `КонвертацияДанныхXDTOСервер` - серверный API движка (используется генератором:
  `ТегиДляМодуляМенеджераОбъекта()` и др.)
- `КонвертацияДанныхXDTOВызовСервера`, `КонвертацияДанныхXDTOКлиент`
- `ОбменДаннымиКонвертацияДанныхXDTO`, `ОбменДаннымиСобытияКонвертацияДанныхXDTO`
- `НастройкиКонвертацииВызовСервера`, `НастройкиКонвертацииПовтИсп`
- `ОбновлениеИнформационнойБазыКонвертацияДанныхXDTO`

Структура процедур генерируемого менеджера (ПКО_<Имя>_ПриОтправкеДанных и т.п.) -
в `format-extension.md`.
