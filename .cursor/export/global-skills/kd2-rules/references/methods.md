# Экспортные методы обработок КД 2.0

Все методы в модулях объектов обработок (`src/DataProcessors/<Имя>/Ext/ObjectModule.bsl`)
в выгрузке `C:\EDT\KD\src\`. Контекст вызова через `execute_code` MCP-toolkit -
подтвержден работающим без явного `execution_context` для всех методов
ниже (toolkit живет в клиентском контексте 1С).

## Выгрузка правил в XML

### `Обработки.ВыгрузкаКонвертации.ВыполнитьВыгрузкуПравил()`

- file: `src/DataProcessors/ВыгрузкаКонвертации/Ext/ObjectModule.bsl:1898`
- Главная точка выгрузки правил конвертации в XML.

#### Реквизиты обработки

| Реквизит | Тип | Назначение |
|---|---|---|
| КонвертацияДляВыгрузки | CatalogRef.Конвертации | **Обязательно** - ссылка на конвертацию |
| ИмяФайлаПравил | string | **Обязательно** - путь к выходному XML (если `СжиматьФайлПравил=true`, расширение `.zip`) |
| СжиматьФайлПравил | boolean | Если `true` - архивирует выход в .zip |
| ВыгружатьПравилаКорреспондента | boolean | Парный обмен (2 конвертации в одном файле) |
| КонвертацияКорреспондентДляВыгрузки | CatalogRef.Конвертации | Если `ВыгружатьПравилаКорреспондента=true` |
| ВыгружатьПравилаРегистрации | boolean | Включить правила регистрации в файл |
| РегистрацияДляВыгрузки | CatalogRef.Регистрации | Если `ВыгружатьПравилаРегистрации=true` |
| ИмяФайлаМодуляВыгрузки | string | Опционально - путь к .txt с модулем обработчиков выгрузки (для отладки) |
| ИмяФайлаМодуляЗагрузки | string | То же для модуля загрузки |
| ИмяФайлаОбработчиковВыгрузки | string | Опционально |
| ИмяФайлаОбработчиковЗагрузки | string | Опционально |
| РежимСохраненияВерсии | decimal | Версия формата (обычно по умолчанию) |

#### Минимальный BSL

```bsl
Конв = Справочники.Конвертации.НайтиПоНаименованию("Имя_Конвертации");
Если Конв.Пустая() Тогда
    ВызватьИсключение "Конвертация не найдена";
КонецЕсли;

Обр = Обработки.ВыгрузкаКонвертации.Создать();
Обр.КонвертацияДляВыгрузки         = Конв;
Обр.ИмяФайлаПравил                 = "C:\Output\rules.xml";
Обр.СжиматьФайлПравил              = Ложь;
Обр.ВыгружатьПравилаКорреспондента = Ложь;
Обр.ВыгружатьПравилаРегистрации    = Ложь;
Обр.ВыполнитьВыгрузкуПравил();

Файл = Новый Файл(Обр.ИмяФайлаПравил);
Результат = Новый Структура("ok,path,size", Истина, Обр.ИмяФайлаПравил, Файл.Размер());
```

## Метаданные конфигураций

### `Обработки.ВыгрузкаМетаданных.ВыполнитьВыгрузку()`

- file: `src/DataProcessors/ВыгрузкаМетаданных/Ext/ObjectModule.bsl:28`
- **Запускается в ИБ источника/приёмника**, не в КД. Создает XML с описанием
  метаданных текущей конфигурации (для последующей загрузки в КД через
  `ЗагрузкаСтруктурыМетаданных`).

#### Реквизиты (через переменные модуля и реквизиты обработки)

| Реквизит | Тип |

| НеВыгружатьРегистрыНакопления | boolean |

Обр.ИмяФайлаВыгрузки = "C:\Output\metadata_source.xml";

Обр.ВыполнитьВыгрузку();

### `Обработки.ЗагрузкаСтруктурыМетаданных.ВыполнитьЗагрузку()`

- file: `src/DataProcessors/ЗагрузкаСтруктурыМетаданных/Ext/ObjectModule.bsl:19`
- Загружает XML метаданных в КД. Создает элемент `Конфигурации` + подчиненные
  `Объекты`/`Свойства`/`Значения`.
- Модуль помечен `#Если Клиент Тогда`, но через MCP-toolkit вызывается без
  проблем (подтверждено: ошибка возникает только если файл не существует -
  значит метод отработал до открытия файла).

#### Параметры (через переменные модуля)

| Переменная | Тип | Назначение |
|---|---|---|
| мЗагрузитьНовую | boolean | true - создать новую Конфигурации, false - обновить существующую |
| мКонфигурация | CatalogRef.Конфигурации | Ссылка на существующую конфигурацию (если `мЗагрузитьНовую=false`) |
| мКомментарий | string | Комментарий к загрузке |
| ИмяФайлаЗагрузки | string | Путь к XML файлу метаданных |

#### Использует `ВыполнятьЗагрузкуСтруктурыВОднойТранзакции` для атомарной загрузки.

#### Минимальный BSL

```bsl
Обр = Обработки.ЗагрузкаСтруктурыМетаданных.Создать();
Обр.ИмяФайлаЗагрузки = "C:\Input\metadata_source.xml";
Обр.мЗагрузитьНовую  = Истина;
Обр.мКомментарий     = "Загрузка БП КОРП v3.0.189.29";
Рез = Обр.ВыполнитьЗагрузку();
Результат = ?(Рез = Ложь, "ошибка загрузки", "ok");
```

## Автонастройка правил

Полезно после загрузки метаданных источника и приёмника - быстро создать
«каркас» правил по совпадающим именам.

| Обработка | Назначение |
|---|---|
| `АвтонастройкаПравилКонвертацииОбъектов` | ПКО по совпадающим именам объектов |
| `АвтонастройкаПравилКонвертацииСвойств` | ПКС внутри ПКО по совпадающим именам реквизитов |
| `АвтонастройкаПравилКонвертацииЗначений` | ПКЗ по совпадающим именам значений |

Конкретные методы и реквизиты - смотри XML обработок в `src/DataProcessors/Автонастройка*/`.
