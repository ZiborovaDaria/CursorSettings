# SB: YAxUnit и тестирование доработок в расширениях

Документ для проекта **SB** (БП 3.0). Общий запуск — в `../SKILL.md`.

## Расширения в репозитории

| Каталог | Имя в ИБ | Назначение |
|---------|----------|------------|
| `Extent/YAXUNIT` | **YAXUNIT** | Движок YAxUnit + **общие модули тестов** (`SB_ОМ_*`) |
| `Extent/Tests` | **SB_Tests** | Оболочка расширения тестов (роль, язык); **BSL-тестов здесь нет** |
| `Extent/SinhUch` | **СинхронизацияОтражениеВУчете** | Тестируемая логика (`СинхУч_*`) |

Префикс имён тестовых модулей в YAXUNIT: **`SB_ОМ_`** (общий модуль, суффикс по области).

## Архитектура (обязательно)

### Где писать тесты

- **Модули тестов** — только в **`Extent/YAXUNIT`**, не в `Extent/Tests` / не в расширении с префиксом доработки.
- **Причина:** отдельное расширение-тест (`SB_Tests`) **не видит** экспортные модули другого расширения (`СинхУч_ОбменУНФСервер`) в глобальном контексте; компиляция падает или тесты не находятся.
- **YAXUNIT** в ИБ подключается вместе с тестируемым расширением — вызовы `СинхУч_*` из `SB_ОМ_*` **работают**.

### Что нельзя в тестовом модуле YAXUNIT

В изолированном расширении YAXUNIT (без полного прикладного контекста КФ) **не использовать**:

| Конструкция | Почему |
|-------------|--------|
| `Документы.*`, `Справочники.*`, `Регистры.*` | Нет глобального контекста метаданных доработки |
| `ОбщегоНазначения`, `ТекущаяДатаСеанса()` | То же / зависимость от SSL вне расширения |
| Создание реальных объектов документов в ИБ | Нет `Документы.РеализацияТоваровУслуг.СоздатьДокумент()` и т.п. |

**Допустимо:** `Неопределено`, `Новый Структура` (заглушка объекта с полями `.Дата`, `.ОбменДанными`), прямые вызовы модулей **подключённого** расширения (`СинхУч_ОбменУНФСервер`), API **ЮТест** / **ЮТТесты**.

### Сценарии, которые не покрывать unit-тестом из YAXUNIT

- Логика, требующая **реального** `ДокументОбъект` с методами платформы (например отмена по **дате запрета** — типовой код обращается к `.Дата` объекта, не к структуре).
- Проведение, запись в регистры, полный обмен с УНФ — **внешняя обработка**, интеграционный прогон в основной КФ или ручная проверка.

Пример удалённого теста: `ТестОтменаПоДатеЗапретаБезДаты` — передача `Неопределено` вместо документа не воспроизводит контракт типовой процедуры.

## Предусловия в InfoBase11

1. **YAXUNIT** — подключено, обновлено из `Extent/YAXUNIT` (F7), **безопасный режим выключен**.
2. **СинхронизацияОтражениеВУчете** — подключено, безопасный режим выключен.
3. **SB_Tests** — для прогона **не обязательно** (пустая оболочка).
4. Платформа в `.tests/settings.json` должна совпадать с ИБ (сейчас **8.5.1.1343**); расхождение с другими сеансами → конфликт версий.

## Конфигурация прогона

| Файл | Назначение |
|------|------------|
| `.tests/settings.json` | `platform`, `infobase`; `user`/`password` пустые (ИБ без пользователей) |
| `.tests/yaxunit-config.json` | `filter.modules`: `["SB_ОМ_СинхУчОбменУНФ"]` — **не** `SB_Tests` |
| `.tests/run-tests.ps1` | ENTERPRISE + `RunUnitTests=…`; перед стартом — `Wait-1cLicenseSlot` |

Запуск:

```powershell
& "C:\Cursor\SB\.tests\run-tests.ps1"
& "C:\Cursor\SB\.tests\run-tests.ps1" -Filter "SB_ОМ_СинхУчОбменУНФ"
```

Отчёт: `.tests/reports/junit.xml`. Успех: exit **0**, в junit — suite `SB_ОМ_СинхУчОбменУНФ`.

## Роли агента и пользователя (SB)

| Действие | Кто |
|----------|-----|
| Правка `Extent/YAXUNIT/.../Module.bsl`, XML модуля | Агент |
| LoadConfigFromFiles + UpdateDBCfg для **YAXUNIT** (и при изменении — **СинхУч**) | **Пользователь** (конфигуратор) |
| `run-tests.ps1` | Агент (после `Wait-1cLicenseSlot`, без других процессов 1С) |
| Отключение безопасного режима расширений | **Пользователь** |

После сообщения пользователя **«обновила»** — сразу прогон `run-tests.ps1`, не повторять загрузку из файлов.

## Диагностика

| Симптом | Действие |
|---------|----------|
| **0 тестов**, пустой отчёт | Журнал 1С (`get_event_log`) — ошибки компиляции модуля; проверить, что модуль в **YAXUNIT** и имя в `filter.modules` |
| Ошибки `Документы`, `ОбщегоНазначения` | Убрать глобальный контекст КФ; заглушки-структуры или `Неопределено` |
| **6 тестов**, 1 error (дата запрета) | Удалить/не добавлять тест без реального документа; перезагрузить YAXUNIT в ИБ |
| Exit 2 | Увеличить `-TimeoutSec`; проверить зависший `1cv8c` |
| Конфликт версий платформы | Закрыть все 1С, выдержать 30 с, одна версия в `settings.json` и конфигураторе |

## Эталон модуля (СинхУч)

Путь: `Extent/YAXUNIT/CommonModules/SB_ОМ_СинхУчОбменУНФ/Ext/Module.bsl`.

- `ИсполняемыеСценарии`: `ДобавитьТест` (клиент) + `ДобавитьСерверныйТест` (сервер).
- Серверные тесты вызывают только `СинхУч_ОбменУНФСервер` с безопасными параметрами.

При добавлении тестов для другого расширения: новый `SB_ОМ_<Имя>` в YAXUNIT, строка в `yaxunit-config.json` → `filter.modules`.
