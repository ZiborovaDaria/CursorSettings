# BSL-шаблоны для execute_code (КД 3.1, toolkit :6011)

Серверный контекст, результат через `Результат`. Кириллицу слать payload-файлом
(`scripts/kd31_exec.sh`), ответ читать из файла через Read. Считать количество -
запросом (не `Количество()`). ПКС - справочник.

Реквизиты в шаблонах выверены по EDT-эталону (`data-schema.md`). Сами операции
записи на живой ИБ НЕ прогонялись (на дату Релизов=0, привязывать не к чему) -
перед применением проверить на тестовой копии КД, снять защиту записи в форме
toolkit, сверить значения перечислений.

КРИТИЧНО по владельцам: ПКО/ПОД/ПКПД - **топ-уровневые** (без Владельца). Связь с
Конвертацией - запись справочника `СоставыКонвертаций` (Владелец=Конвертация,
`ЭлементКонвертации`=ссылка на правило). Только **ПКС** подчинён: Владелец=ПКО.

## Инспекция структуры (надёжный однострочный паттерн)

```bsl
С = ""; Для Каждого М Из Метаданные.Справочники Цикл С = С + М.Имя + ","; КонецЦикла; Результат = С;
```

Реквизиты объекта:

```bsl
С = ""; Для Каждого Р Из Метаданные.Справочники.ПравилаКонвертацииОбъектов.Реквизиты Цикл С = С + Р.Имя + ","; КонецЦикла; Результат = С;
```

Сохранять ответ в файл (`kd31_exec.sh script.bsl out.json`) и читать через Read.

```sql
ВЫБРАТЬ КОЛИЧЕСТВО(*) КАК К ИЗ Справочник.Конвертации
```

```sql
       Конфигурация, КонфигурацияКорреспондент, ВерсияФорматаМенеджера

ГДЕ НЕ ЭтоГруппа
```

## Правила, входящие в конвертацию (через СоставыКонвертаций)

ПКО/ПОД/ПКПД не имеют Владельца - их связь с конвертацией идёт через
`СоставыКонвертаций`. `ЭлементКонвертации` - составной тип (ПКО/ПОД/ПКПД/Алгоритмы/Запросы).

```sql
ВЫБРАТЬ Состав.ЭлементКонвертации КАК Правило, Состав.Отключить
ИЗ Справочник.СоставыКонвертаций КАК Состав
ГДЕ Состав.Владелец.Наименование = &Конв
```

Только ПКО (фильтр по типу значения ссылки):

```sql
ВЫБРАТЬ Состав.ЭлементКонвертации КАК ПКО
ИЗ Справочник.СоставыКонвертаций КАК Состав
ГДЕ Состав.Владелец.Наименование = &Конв
       И ВЫРАЗИТЬ(Состав.ЭлементКонвертации КАК Справочник.ПравилаКонвертацииОбъектов) ЕСТЬ НЕ NULL
```

Свойства ПКС читаются по Владельцу=ПКО (ПКС иерархичен - ЭтоГруппа есть; ПКО НЕ
иерархичен - у ПКО ЭтоГруппа НЕТ):

```sql
ВЫБРАТЬ ПКС.СвойствоКонфигурации, ПКС.СвойствоФормата, ПКС.Поиск, ПКС.ПравилоКонвертацииСвойства
ИЗ Справочник.ПравилаКонвертацииСвойств КАК ПКС
ГДЕ ПКС.Владелец = &ПКО
```

## Создание ПКО (топ-уровневый) и включение в конвертацию

ВАЖНО (эталон демо КД 3.1, конвертация БП->УП): у ПКО **НЕТ реквизита Наименование**
и ПКО **НЕ иерархичен** (нет ЭтоГруппа). Идентификатор правила хранится в **Коде**
(`Справочник_Номенклатура_Отправка`). Искать ПКО по Коду (НайтиПоКоду), не по
Наименованию. Паттерн: на объект создаётся **2 ПКО - отдельно _Отправка и
_Получение** (НЕ одно ПКО на оба направления):
- `<Объект>_Отправка`: ИспользоватьДляОтправки=Истина.
- `<Объект>_Получение`: ИспользоватьДляПолучения=Истина, ВариантИдентификации=ПоУникальномуИдентификатору.

Предусловие: загружен Релиз (есть Объекты), загружен формат (есть ОбъектыФормата).
`Объекты` подчинены `Релизы` - искать с Владельцем=Релиз (`Конв.Конфигурация`).

```bsl
Конв = Справочники.Конвертации.НайтиПоНаименованию("Демо_РасширениеEnterpriseData");
ОбъКонф = Справочники.Объекты.НайтиПоНаименованию("СправочникСсылка.Номенклатура", Истина, , Конв.Конфигурация);

// ПКО _Отправка - топ-уровневый, идентификатор в Коде (Наименования НЕТ)
ПКО = Справочники.ПравилаКонвертацииОбъектов.СоздатьЭлемент();
ПКО.Код = "Справочник_Номенклатура_Отправка";    // идентификатор правила
ПКО.ОбъектКонфигурации = ОбъКонф;                 // CatalogRef.Объекты (НЕ строка)
ПКО.ОбъектФормата = "Справочник.Номенклатура";    // String (имя объекта формата)
ПКО.ИспользоватьДляОтправки = Истина;
ПКО.ТипКонвертации = Перечисления.ТипыКонвертаций.XDTO;
ПКО.Записать();

// ПКО _Получение - отдельная запись
ПКОПолуч = Справочники.ПравилаКонвертацииОбъектов.СоздатьЭлемент();
ПКОПолуч.Код = "Справочник_Номенклатура_Получение";
ПКОПолуч.ОбъектКонфигурации = ОбъКонф;
ПКОПолуч.ОбъектФормата = "Справочник.Номенклатура";
ПКОПолуч.ИспользоватьДляПолучения = Истина;
ПКОПолуч.ТипКонвертации = Перечисления.ТипыКонвертаций.XDTO;
ПКОПолуч.ВариантИдентификации = Перечисления.ВариантыИдентификацииОбъектов.ПоУникальномуИдентификатору;
ПКОПолуч.Записать();

// Включение каждого ПКО в конвертацию - запись СоставыКонвертаций (Владелец=Конвертация)
Сост = Справочники.СоставыКонвертаций.СоздатьЭлемент();
Сост.Владелец = Конв;
Сост.ЭлементКонвертации = ПКО.Ссылка;   // составной тип; аналогично для ПКОПолуч
Сост.Записать();

Результат = XMLСтрока(ПКО.Ссылка);
```

Значения `ВариантыИдентификацииОбъектов`: `ПоУникальномуИдентификатору`,
`ПоПолямПоиска`, `СначалаПоУникальномуИдентификаторуПотомПоПолямПоиска`.
`ТипыКонвертаций`: `XML`, `XDTO`.

## Создание ПКС (Владелец=ПКО)

Внимание к типам: `СвойствоКонфигурации` - ссылка (CatalogRef.Свойства), а
`СвойствоФормата` и `ПравилоКонвертацииСвойства` - **строки**.
`ПравилоКонвертацииСвойства` для ссылочного свойства = **Код целевого ПКО строкой**
(напр. "Справочник_Контрагенты", "Перечисление_СтавкиНДС") - эталон демо.

Паттерн демо: в ПКО _Отправка часть ПКС идёт БЕЗ СвойствоКонфигурации (null) +
ИспользуетсяАлгоритмКонвертации=Истина (значение кладёт алгоритм ПКО ПриОтправкеДанных).
Прямые ПКС - со ссылкой СвойствоКонфигурации.

```bsl
СвКонф = Справочники.Свойства.НайтиПоНаименованию("Артикул", Истина, , ОбъКонф);

// Прямое ПКС: свойство конфигурации <-> свойство формата
ПКС = Справочники.ПравилаКонвертацииСвойств.СоздатьЭлемент();
ПКС.Владелец = ПКО.Ссылка;              // ПКС подчинён ПКО
ПКС.СвойствоКонфигурации = СвКонф;       // CatalogRef.Свойства
ПКС.СвойствоФормата = "Артикул";         // String (имя свойства формата)
ПКС.Поиск = Ложь;                        // Истина - поле поиска
// ссылочное свойство: ПКС.ПравилоКонвертацииСвойства = "Справочник_Контрагенты"; // Код ПКО строкой
ПКС.Записать();
Результат = XMLСтрока(ПКС.Ссылка);
```

Алгоритмическое ПКС (значение формирует алгоритм, СвойствоКонфигурации не задаём):

```bsl
ПКСа = Справочники.ПравилаКонвертацииСвойств.СоздатьЭлемент();
ПКСа.Владелец = ПКО.Ссылка;
ПКСа.СвойствоФормата = "Демо_СтатусИзУНФ";          // только свойство формата
ПКСа.ИспользуетсяАлгоритмКонвертации = Истина;
ПКСа.АлгоритмПередВыгрузкойСвойства = "Значение = ДанныеИБ.Демо_СтатусИзУНФ;";
ПКСа.Записать();
```

Алгоритмы ПКС (строки BSL): `АлгоритмПередВыгрузкойСвойства`,
`АлгоритмПриВыгрузкеСвойства`, `АлгоритмПослеВыгрузкиСвойства`,
`АлгоритмПередОбработкойВыгрузки`, `АлгоритмПослеОбработкиВыгрузки`.

## Привязка версии формата (в т.ч. расширения) к Конвертации

В ТЧ `Конвертации.ВерсииФормата` единственный реквизит - `ВерсияФормата`
(ссылка на Справочник.ВерсииФормата).

```bsl
ВерсияРасш = Справочники.ВерсииФормата.НайтиПоНаименованию("Расширение Демо 1.0");
Об = Конв.ПолучитьОбъект();
Найдено = Ложь;
Для Каждого Стр Из Об.ВерсииФормата Цикл
    Если Стр.ВерсияФормата = ВерсияРасш Тогда Найдено = Истина; КонецЕсли;
КонецЦикла;
Если НЕ Найдено Тогда
    НоваяСтр = Об.ВерсииФормата.Добавить();
    НоваяСтр.ВерсияФормата = ВерсияРасш;
    Об.Записать();
КонецЕсли;
Результат = "ok";
```

## Загрузка структуры конфигурации headless (ПРОВЕРЕНО, работает)

Форма `ЗагрузкаСтруктурыКонфигурацииИзФайловXML` собирает файлы и запускает фон.
Реальный парсер - экспортный метод менеджера, вызывается из `execute_code`
(headless подтверждён на УНФ ~17к .mdo, ~31 мин: Релиз+Объекты=4060, Свойства=69015):

```
Обработки.ЗагрузкаСтруктурыКонфигурацииИзФайловXML.ВыполнитьЗагрузкуМетаданных(СтруктураПараметров, АдресРезультата)
```

`СтруктураПараметров` (метод формы `ВыполнитьЗагрузкуСервер`): `СпособЗагрузки`
(0=новый Релиз), `ПомещенныеФайлы` (массив структур `{ПолноеИмя, Хранение=путь к
файлу на сервере, ЭтоРасширение}`), `ИмяНовойКонфигурации`, `Конфигурация`,
`ИсточникДанных`, `ВариантЗагрузкиДвижений`, `ТолькоОбновитьПланыОбмена`,
`ТолькоДобавлятьНовые`, `ЕстьРасширения`, `ИмяКаталогаЗагрузки`,
`КаталогиРасширений` (если есть расширения). Поддерживается формат EDT напрямую
(ридеры `ПрочитатьСведенияОбОбъектеИзEDT`/`ПрочитатьСведенияОСвойствахИзEDT`).

ЗАГРУЗКА С РАСШИРЕНИЕМ КОНФИГУРАЦИИ: ЕстьРасширения=Истина; у .mdo расширения в
ПомещенныеФайлы ставить ЭтоРасширение=Истина; КаталогиРасширений=[каталог расширения].
Без этого Демо-реквизиты не попадут в теневую модель (Справочник.Свойства) и ПКС
для них не создать.

ГРАБЛЯ: загрузчик в конце делает `УдалитьФайлы(Хранение)` по каждому файлу -
грузить из ВРЕМЕННОЙ копии (robocopy), НЕ из EDT-оригиналов (снесёт исходники).

Большая конфигурация -> фоновое задание (`ДлительныеОперации.ВыполнитьВФоне`) с
опросом состояния. Опрос НЕ по счётчикам (объекты стабилизируются задолго до конца -
идёт длинная фаза заполнения типов), а по `ФоновыеЗадания.НайтиПоУникальномуИдентификатору(JobID).Состояние`.
Делать на тестовой копии КД. Детали - `methods.md`.

## Загрузка базового формата EnterpriseData headless (ПРОВЕРЕНО)

`Обработки.ЗагрузкаСтруктурыФормата.ВыполнитьЗагрузкуФормата(СП, Адрес)` - строит
фабрику XDTO из XSD-ФАЙЛОВ и создаёт ОбъектыФормата/СвойстваФормата/ЗначенияФормата.
Нужен XSD стандартного вида (`xs:schema`), НЕ внутренний `Package.xdto` EDT.
Источник XSD: выгрузить XDTO-пакет EnterpriseData из УНФ через Конфигуратор
(XDTO-пакет -> Выгрузить в файл XSD) + вспомогательный ExchangeMessage.xsd.

```bsl
ДанныеДляЗагрузки = Новый Массив;
ДанныеДляЗагрузки.Добавить("E:\...\ExchangeMessage.xsd");        // вспомогательный пакет Message
ДанныеДляЗагрузки.Добавить("E:\...\EnterpriseData_1_22_1.xsd");  // основной (с xs:import первым)
СП = Новый Структура;
СП.Вставить("СпособЗагрузки", 0);
СП.Вставить("ВерсияФормата", Справочники.ВерсииФормата.ПустаяСсылка());
СП.Вставить("ДобавлятьТолькоНовые", Ложь);
СП.Вставить("ЭтоРасширение", Ложь);                              // Истина - для XSD расширения формата
СП.Вставить("ИмяОсновногоПакетаXDTO", "http://v8.1c.ru/edi/edi_stnd/EnterpriseData/1.22");
СП.Вставить("РодительВерсии", Справочники.ВерсииФормата.ПустаяСсылка());
СП.Вставить("ШаблонПространстваИмен", "http://v8.1c.ru/edi/edi_stnd/EnterpriseData");
СП.Вставить("НомерВерсииФормата", "1.22");
СП.Вставить("ДанныеДляЗагрузки", ДанныеДляЗагрузки);
Адрес = ПоместитьВоВременноеХранилище(Неопределено);
Обработки.ЗагрузкаСтруктурыФормата.ВыполнитьЗагрузкуФормата(СП, Адрес);
Рез = ПолучитьИзВременногоХранилища(Адрес);
Результат = "Успех=" + Строка(Рез.Успех);
```

Распознавание основного пакета: загрузчик считает ОСНОВНЫМ тот XSD, где сразу за
`xs:schema` идёт `xs:import` (импорт Message). targetNamespace этого файла =
`ИмяОсновногоПакетаXDTO`. XSD расширения формата (namespace Демо) грузить отдельно
с `ЭтоРасширение=Истина` - создаст связь base<->расширение в
РегистрСведений.ВзаимосвязиВерсийФорматовИРасширений.

ГРАБЛЯ Gateway Timeout: тяжёлый синхронный execute_code (формат ~1.2МБ XSD) даёт
HTTP `{"error":"Gateway Timeout"}` от прокси, НО на стороне 1С операция
ЗАВЕРШАЕТСЯ. Проверять РЕЗУЛЬТАТ отдельным запросом (КОЛИЧЕСТВО ОбъектыФормата),
НЕ доверять HTTP-ответу. Не запускать повторно «на всякий случай» - создаст дубль/ошибку.
