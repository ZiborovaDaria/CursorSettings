Используй только если есть реальный выбор: архитектура, способ расширения, hook, структура EPF/ERF, интеграция, производительность, UI формы.

Шаблон:

1. Problem.
2. Constraints.
3. Options A/B/C.
4. Trade-offs.
5. Decision.
6. Implementation notes.
7. Verification.
8. Rollback.

Сохрани результат в `memory-bank/creative/creative-YYYY-MM-DD-<topic>.md`.
После решения переходи к `/implement`.
