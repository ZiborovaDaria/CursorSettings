# YAxUnit Runner — кросс-проектный запуск тестов

## Когда использовать

- Пользователь просит «запусти тесты», «прогони тесты», «проверь тестами»
- После написания или изменения тестового модуля
- Для валидации доработки в **расширении** (см. раздел «Тесты в расширениях»)
- После «обновила» / F7 в конфигураторе — **только** `run-tests.ps1`, без повторной загрузки CFE

## Архитектура (в каждом проекте)

| Компонент | Путь | Назначение |
|---|---|---|
| Настройки | `.tests/settings.json` | Пути к ИБ, платформе, учётные данные |
| Конфиг YAxUnit | `.tests/yaxunit-config.json` | JSON: `filter.modules` / `filter.tests`, отчёт |
| Скрипт | `.tests/run-tests.ps1` | PowerShell-обёртка + `Wait-1cLicenseSlot` |
| Отчёты | `.tests/reports/*.xml` | JUnit XML после прогона |

## Запуск 1С (лицензии)

Перед **любым** стартом `1cv8` / `1cv8c` / DESIGNER / ENTERPRISE:

1. **Один экземпляр** — дождаться завершения предыдущего процесса 1С.
2. **Пауза 30 секунд** между последовательными запусками (в т.ч. DESIGNER → тесты).

См. **`@rules/1c-single-instance-launch.mdc`**. Скрипт: `.cursor/scripts/Wait-1cLicenseSlot.ps1` (вызывается из `run-tests.ps1`).

**Запрещено** в одной цепочке PowerShell без ожидания: LoadConfigFromFiles → UpdateDBCfg → ENTERPRISE.

## Запуск

```powershell
& "<ProjectRoot>\.tests\run-tests.ps1"
& "<ProjectRoot>\.tests\run-tests.ps1" -Filter "ИмяМодуля"
& "<ProjectRoot>\.tests\run-tests.ps1" -Filter "ИмяМодуля.ИмяТеста"
& "<ProjectRoot>\.tests\run-tests.ps1" -ShowUI
& "<ProjectRoot>\.tests\run-tests.ps1" -TimeoutSec 600
```

## Чтение результатов

1. **stdout** — сводка: Total / Passed / Failed / Errors / Skipped
2. **Exit code**: `0` = OK, `1` = есть падения, `2` = timeout
3. Детали — JUnit XML в `.tests/reports/`

Дымовой тест на клиенте + те же сценарии на сервере дают **дубли** в счётчике (например 6 записей при 4 уникальных методах) — это норма YAxUnit.

## Роли агента и пользователя

| Действие | Кто |
|---|---|
| Правка BSL/XML тестов в репозитории | Агент |
| LoadConfigFromFiles + UpdateDBCfg в ИБ | **Пользователь** |
| Запуск `run-tests.ps1` | Агент |
| Анализ отчёта, исправление упавших тестов | Агент |
| Отключение безопасного режима расширений | **Пользователь** |

## Тесты в расширениях (критично)

Отдельное расширение «только тесты» (**`<Проект>_Tests`**) **не вызывает** экспортные модули другого расширения (нет общего глобального контекста между CFE).

| Подход | Когда |
|--------|--------|
| Тесты в **том же** расширении, что и код | Малые CFE без YAxUnit в ИБ |
| Тесты в расширении **YAXUNIT** рядом с движком | **SB и аналоги** — вызовы `Префикс_Модуль` тестируемого CFE |

В модуле тестов **внутри YAXUNIT** (изолированное расширение) **нельзя**: `Документы`, `Справочники`, `Регистры`, `ОбщегоНазначения`, `ТекущаяДатаСеанса`, создание реальных объектов в ИБ.

**Можно:** `Неопределено`, `Новый Структура` как заглушка полей, вызовы экспортных модулей **подключённого** расширения, **ЮТест** / **ЮТТесты**.

Сценарии с реальным `ДокументОбъект`, проведением, датой запрета — **не** unit-тест из YAXUNIT.

## Структура тестового модуля

```bsl
#Область ПрограммныйИнтерфейс

Процедура ИсполняемыеСценарии() Экспорт

    ЮТТесты
        .ДобавитьТест("ТестИмяМетода")           // клиент
        .ДобавитьСерверныйТест("ТестНаСервере"); // сервер

КонецПроцедуры

#КонецОбласти

#Область СлужебныеПроцедурыИФункции

Процедура ТестИмяМетода() Экспорт

    ЮТест.ОжидаетЧто(Истина)
        .Равно(Истина);

КонецПроцедуры

#КонецОбласти
```

После добавления модуля в YAXUNIT: XML в `CommonModules/`, регистрация в `Configuration.xml` / `ConfigDumpInfo.xml`, строка в `filter.modules`.

## Настройки проектов (.tests/settings.json)

```json
{
  "platform": "C:\\Program Files\\1cv8\\8.5.1.1343\\bin\\1cv8c.exe",
  "infobase": "<путь к файловой ИБ>",
  "user": "",
  "password": ""
}
```

Поле `extensionName` в SB **не используется** скриптом прогона; актуален каталог **YAXUNIT** и `yaxunit-config.json`.

## Предварительные условия (для каждого проекта)

1. Расширение **YAxUnit** подключено к ИБ, **безопасный режим отключён**
2. Тестируемое расширение подключено и обновлено
3. Версия `1cv8c` в `settings.json` совпадает с конфигуратором (иначе конфликт версий)
4. Для SB: см. **`docs/sb-extension-tests.md`**

## Диагностика (общая)

| Симптом | Проверка |
|---------|----------|
| 0 тестов | Компиляция модуля; `filter.modules`; журнал 1С |
| Ошибки `Документы` / `ОбщегоНазначения` | Убрать глобальный контекст КФ из теста |
| Exit 2 | Timeout, зависший процесс 1С |
| 0 после правок в git | Пользователь не обновил YAXUNIT в ИБ — напомнить F7 |
