# /implement — реализация

Совместимый режим со старым Supercode Memory Bank. `/build` является алиасом.

1. Прочитай `tasks.md`, `activeContext.md`, последний plan/creative.
2. Не выходи за scope.
3. Если задача требует код — загрузи `1c-code-writing-agent.mdc` и детальное правило.
4. Для ESTI-кода используй `project-esti-mcp-router-agent.mdc`.
5. Найди типовой аналог перед написанием кода.
6. Внеси минимальные изменения.
7. Пиши реальный BSL/XML/PowerShell, не псевдокод, если хватает контекста.
8. Выполни доступную проверку.
9. При ошибке используй `capture-error` / error-learning.
10. Обнови `progress.md`.
11. Подготовь отчет: что изменено, где, как проверить, риски.
