# Схема итогового XML правил обмена

Получена выгрузкой реальной конвертации «БП КОРП -> УПП» через
`Обработки.ВыгрузкаКонвертации.ВыполнитьВыгрузкуПравил()`. Файл 543 КБ,
формат BSP 2.01.

## Корневая структура

```xml
<ПравилаОбмена>
    <ВерсияФормата РежимСовместимости="РежимСовместимостиСБСП20">2.01</ВерсияФормата>
    <Ид>00000000-0000-0000-0000-000000000001    </Ид>
    <Наименование>EXCH-0000001 (2025) - БП КОРП -> УПП</Наименование>
    <ДатаВремяСоздания>2026-05-20T19:53:37</ДатаВремяСоздания>
    <Источник ВерсияПлатформы="8.0" ВерсияКонфигурации="3.0.189.29"
              СинонимКонфигурации="Бухгалтерия предприятия КОРП, редакция 3.0">
        БухгалтерияПредприятияКОРП
    </Источник>
    <Приемник ВерсияПлатформы="8.0" ВерсияКонфигурации=""
              СинонимКонфигурации="">
        УправлениеПроизводственнымПредприятием
    </Приемник>
    <Параметры/>
    <Обработки/>
    <ПравилаКонвертацииОбъектов>...</ПравилаКонвертацииОбъектов>
    <ПравилаВыгрузкиДанных>...</ПравилаВыгрузкиДанных>
    <ПравилаОчистки>...</ПравилаОчистки>
    <Алгоритмы>...</Алгоритмы>
    <Запросы>...</Запросы>
</ПравилаОбмена>
```

## Маппинг полей справочников на XML

### Конвертации -> корневые теги

| Поле Справочник.Конвертации | XML |
|---|---|
| `Код` | `<Ид>` (с trailing-пробелами до 40 символов) |
| `Наименование` | `<Наименование>` |
| `РежимСовместимости` | атрибут `РежимСовместимости` в `<ВерсияФормата>` |
| - (константа) | `<ВерсияФормата>2.01</ВерсияФормата>` |
| (текущая дата при выгрузке) | `<ДатаВремяСоздания>` |

### Конфигурации (Источник/Приемник) -> атрибуты

| Поле Справочник.Конфигурации | XML атрибут |
|---|---|
| `Имя` | текст элемента `<Источник>` или `<Приемник>` |
| `Версия` | атрибут `ВерсияКонфигурации` |
| `Синоним` | атрибут `СинонимКонфигурации` |
| (фиксировано) | атрибут `ВерсияПлатформы="8.0"` |

## ПКО (`<Правило>` внутри `<ПравилаКонвертацииОбъектов>`)

```xml
<ПравилаКонвертацииОбъектов>
    <Группа>
        <Код>Справочники</Код>
        <Наименование>Справочники</Наименование>
        <Порядок>50</Порядок>
        <Правило>
            <Код>Валюты</Код>
            <Наименование>Справочник: Валюты</Наименование>
            <Порядок>50</Порядок>
            <НеЗамещать>true</НеЗамещать>
            <НеВыгружатьОбъектыСвойствПоСсылкам>true</НеВыгружатьОбъектыСвойствПоСсылкам>
            <НеСоздаватьЕслиНеНайден>true</НеСоздаватьЕслиНеНайден>
            <Источник>СправочникСсылка.Валюты</Источник>
            <Приемник>СправочникСсылка.Валюты</Приемник>
            <Свойства>...</Свойства>
            <Значения/>
        </Правило>
    </Группа>
</ПравилаКонвертацииОбъектов>
```

### Маппинг ПКО

| Поле Справочник.ПравилаКонвертацииОбъектов | XML |
|---|---|
| `Наименование` (ПКО-группа с `ЭтоГруппа=true`) | контейнер `<Группа>` |
| Обычная ПКО (`ЭтоГруппа=false`) | элемент `<Правило>` |
| `Код` (имя для XML, не Наименование) | `<Код>` |
| `Наименование` | `<Наименование>` |
| `Порядок` | `<Порядок>` |
| `ТипИсточника` | `<Источник>` (строкой, например `ДокументСсылка.АвансовыйОтчет`) |
| `ТипПриемника` | `<Приемник>` |
| `НеЗамещать=true` | `<НеЗамещать>true</НеЗамещать>` (false - не выводится) |
| `НеВыгружатьОбъектыСвойствПоСсылкам=true` | то же |
| `НеСоздаватьОбъектВПриемникеЕслиНеНайденПоПолямПоиска=true` | `<НеСоздаватьЕслиНеНайден>` (имя в XML укорочено) |

## ПКС (`<Свойство>` внутри `<Свойства>`)

```xml
<Свойства>
    <Свойство Поиск="true">
        <Код>2</Код>
        <Наименование>Доп_НСИ_ГУИД --> Доп_НСИ_ГУИД</Наименование>
        <Порядок>50</Порядок>
        <Источник Имя="Доп_НСИ_ГУИД" Вид="Реквизит" Тип="Строка"/>
        <Приемник Имя="Доп_НСИ_ГУИД" Вид="Реквизит" Тип="Строка"/>
    </Свойство>
    <Свойство>
        <Код>1</Код>
        <Наименование>Наименование --> Наименование</Наименование>
        <Порядок>100</Порядок>
        <Источник Имя="Наименование" Вид="Свойство" Тип="Строка"/>
        <Приемник Имя="Наименование" Вид="Свойство" Тип="Строка"/>
    </Свойство>
    <Свойство>
        <Код>2</Код>
        <Наименование>Владелец --> Владелец</Наименование>
        <Порядок>150</Порядок>
        <Источник Имя="Владелец" Вид="Свойство" Тип="СправочникСсылка.Контрагенты"/>
        <Приемник Имя="Владелец" Вид="Свойство" Тип="СправочникСсылка.Контрагенты"/>
        <КодПравилаКонвертации>Контрагенты                                       </КодПравилаКонвертации>
    </Свойство>
</Свойства>
```

### Маппинг ПКС

| Поле Справочник.ПравилаКонвертацииСвойств | XML |
|---|---|
| `Поиск=true` | атрибут `Поиск="true"` на `<Свойство>` (false - атрибут не выводится) |
| (порядковый номер ПКС внутри ПКО) | `<Код>` (генерируется при выгрузке - не путать с реквизитом Код) |
| `Наименование` | `<Наименование>` |
| `Порядок` | `<Порядок>` |
| `ИмяИсточника`, `ВидИсточника`, `ТипИсточника` | атрибуты `Имя`/`Вид`/`Тип` в `<Источник>` |
| `ИмяПриемника`, `ВидПриемника`, `ТипПриемника` | атрибуты в `<Приемник>` |
| `ПравилоКонвертации` (для ссылочного типа) | `<КодПравилаКонвертации>` (Код связанной ПКО с trailing-пробелами до 50 символов) |

### Значения атрибута `Вид`

| Значение | Назначение |
|---|---|
| `Реквизит` | Реквизит объекта (справочника, документа) |
| `Свойство` | Стандартное свойство (Наименование, Код, Дата, Номер) |
| `ТабличнаяЧасть` | Табличная часть |

## ПКЗ (`<Значения>` внутри `<Правило>`)

Структура (для перечислений):

```xml
<Значения>
    <Значение>
        <Источник>ЗначениеПеречисленияИсточника</Источник>
        <Приемник>ЗначениеПеречисленияПриёмника</Приемник>
    </Значение>
</Значения>
```

В исследованной конвертации `<Значения/>` пустые - конвертация на 100%
справочников без перечислений.

## Алгоритмы и Запросы

`<Алгоритмы>` и `<Запросы>` содержат именованные BSL-фрагменты и тексты
запросов, которые потом используются в алгоритмах событий ПКО/ПКС/Конвертации.
В исследованной конвертации они пустые.

## Особенности

### Trailing-пробелы в Код

Реквизит `Код` в Конвертации (40 символов) и в ПКО (50 символов) - строки
фиксированной длины с trailing-пробелами:

```
<Ид>00000000-0000-0000-0000-000000000001    </Ид>
<КодПравилаКонвертации>Контрагенты                                       </КодПравилаКонвертации>
```

При сравнении со ссылкой на КодПравилаКонвертации использовать `СокрЛП()`.

### `<НеСоздаватьЕслиНеНайден>` vs `НеСоздаватьОбъектВПриемникеЕслиНеНайденПоПолямПоиска`

Имя в XML укорочено по сравнению с именем реквизита справочника. Это пример
несоответствия - всегда сверяйся с реальным маппингом, а не с именем реквизита.

### Порядок групп в XML

В корне `<ПравилаОбмена>` группы идут в фиксированном порядке:
1. `<ПравилаКонвертацииОбъектов>`
2. `<ПравилаВыгрузкиДанных>`
3. `<ПравилаОчистки>` (в реквизите справочника называется `ПравилаОчисткиДанных`)
4. `<Алгоритмы>`
5. `<Запросы>`
