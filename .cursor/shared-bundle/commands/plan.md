# /plan — планирование

1. Прочитай `tasks.md`, `activeContext.md`, `projectbrief.md`, `techContext.md`.
2. Проверь факты через MCP/файлы, если задача по коду/метаданным.
3. Раздели план на минимальные шаги.
4. Укажи, какие правила/skills нужны.
5. Укажи проверки после реализации.
6. Укажи риски и rollback.
7. Обнови `memory-bank/tasks.md` и `progress.md`.
8. Если есть архитектурный выбор — отправь в `/creative`.
9. Если выборов нет — переходи к `/implement`.
