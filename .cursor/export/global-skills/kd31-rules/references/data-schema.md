# Объектная модель КД 3.1 (выверено по EDT)

Источник истины: EDT-проект `КД_31_Демопремьер_77УНФ` (конфигурация
«Конвертация данных, редакция 3.1» 3.1.6.14, платформа 8.5.1.1302). Имена,
**типы** реквизитов, владельцы и табличные части получены через EDT MCP
(`get_metadata_details full`). Это эталон, заменивший прежние данные из toolkit
(toolkit искажал кириллицу и не давал типы).

Модель КД 3.1 близка к КД 2.0 (теневая модель метаданных + справочники правил),
плюс слой формата EnterpriseData. Результат разработки - не XML-правила, а модуль
менеджера обмена (BSL) + версия/пакет формата (XSD). XML-правила для «Универсального
обмена» - это КД 2.0 (скилл `kd2-rules`).

КРИТИЧНО: ПКО/ПОД/ПКПД - **топ-уровневые, без Владельца**. С конкретной
Конвертацией связываются записью справочника `СоставыКонвертаций`
(Владелец=Конвертации, реквизит `ЭлементКонвертации` - ссылка на правило). Только
ПКС подчинён - его Владелец=ПКО. У стандартного реквизита Владелец у ПОД остался
синоним «Конвертация», но элемент `<owners>` в .mdo не задан - это рудимент, а не
реальная подчинённость.

ПКС - **отдельный справочник** (как в КД 2.0), не регистр сведений. Обращение
`Метаданные.РегистрыСведений.ПравилаКонвертацииСвойств` падает; правильно
`Метаданные.Справочники.ПравилаКонвертацииСвойств`.

## Справочник.Конвертации (топ, иерарх.)

Пара обмена. Реквизиты (тип):
- `ТипКонвертации` - EnumRef.ТипыКонвертаций (значения: `XML`, `XDTO`)
- `ИнтерфейсКонвертации` - EnumRef.ВариантыМоделейОбмена (синоним «Интерфейс по
  модели»; значения `ПакетнаяМодель`/`ОбъектнаяМодель`/`КомбинированнаяМодель`)
- `Конфигурация`, `КонфигурацияКорреспондент` - **CatalogRef.Релизы** (не Конфигурации!)
- `Статус` - EnumRef.СостоянияПравил (`Черновик`/`ВРаботе`/`НаТестировании`/`Готово`/`Архив`)
- `Версия82` - Boolean; `ВерсияФорматаМенеджера` - String; `ВидБазы` - Number
- Подключение корреспондента: `ИмяБазыДанных`, `СерверБазыДанных`,
  `ПользовательБазыДанных`, `ПарольПользователя`, `ПолноеИмяФайловойБазы` - String
- `ИспользоватьРазделениеМодуля` - Boolean; `СпособВыгрузкиРазделенногоМодуля` -
  EnumRef.СпособыВыгрузкиРазделенногоМодуля
- `ИспользоватьТипизациюПеременных`, `ГруппироватьПоОбъектамВыгрузки`,
  `ЗапретитьПереводИменОбъектовФормата` - Boolean
- `ИмяФайлаДляСохранения`, `Описание` - String
- ~19 алгоритмов-обработчиков (String): `АлгоритмПередВыгрузкойДанных`,
  `АлгоритмПередКонвертацией`, `АлгоритмПослеЗагрузкиДанных`,
  `АлгоритмПослеЗагрузкиПравилОбмена` и т.д.

Табличные части:
- `ВерсииФормата` - реквизит **только** `ВерсияФормата` (CatalogRef.ВерсииФормата).
  (Реквизитов `ОбъектыСохраненыВоВнешнемФайле`/`ИмяВнешнегоФайла` в этой ТЧ нет.)
- `Параметры` - `Параметр`, `ОписаниеПараметра`, `УстанавливатьВДиалоге`,
  `ИспользуетсяПриЗагрузке`, `ПередаватьПараметрПриВыгрузке`, `ТипОбъекта`
  (CatalogRef.Объекты), `ПравилоКонвертации` (String), `ВременнаяКопия`.

`Конфигурация`/`КонфигурацияКорреспондент` пусты, пока не загружен Релиз.

## Справочник.СоставыКонвертаций (Владелец=Конвертации)

Связывает Конвертацию с её правилами и алгоритмами.
- `ЭлементКонвертации` - составной тип: CatalogRef из
  `ПравилаКонвертацииПредопределенныхДанных`, `ПравилаКонвертацииОбъектов`,
  `Алгоритмы`, `Запросы`, `ПравилаОбработкиДанных`.
- `Отключить` - Boolean.

Чтобы правило (ПКО/ПОД/ПКПД) попало в конвертацию, создаётся запись
`СоставыКонвертаций` с Владелец=Конвертация и ЭлементКонвертации=ссылка на правило.

## Справочник.ПравилаКонвертацииОбъектов (ПКО, топ-уровневый)

Код (String, 50) = идентификатор правила (представление по коду). Реквизиты:
- `ОбъектКонфигурации`, `ОбъектКонфигурацииКорреспондент` - CatalogRef.Объекты
- `ОбъектКонфигурацииДляВыгрузкиМодуля` - CatalogRef.Объекты
- `ОбъектФормата` - String (имя объекта формата)
- `ВариантИдентификации` - EnumRef.ВариантыИдентификацииОбъектов
  (`ПоУникальномуИдентификатору`/`ПоПолямПоиска`/
  `СначалаПоУникальномуИдентификаторуПотомПоПолямПоиска`)
- `ТипКонвертации` - EnumRef.ТипыКонвертаций
- `ПриоритетОбъектовОбмена` - EnumRef.ПриоритетОбъектовОбмена
- `ОтборСобытийXDTOПоМодели` - EnumRef.ВариантыМоделейОбмена
- `ДополнительныйМодуль` - CatalogRef.ДополнительныеМодули
- `ГруппаПравил` - String (НЕ ссылка; справочника `ГруппыПравил` нет)
- Флаги (Boolean): `ИспользоватьДляОтправки`, `ИспользоватьДляПолучения`,
  `ВыгружатьВМодульМенеджера`, `ВыгружатьОбъектТолькоПриНаличииНаНегоСсылки`,
  `ГенерироватьНовыйНомерИлиКодЕслиНеУказан`, `ИспользоватьБыстрыйПоискПриЗагрузке`,
  `НеВыгружатьОбъектыСвойствПоСсылкам`, `НеЗамещать`,
  `НеЗамещатьОбъектСозданныйВИнформационнойБазеПриемнике`, `НеЗапоминатьВыгруженные`,
  `НеСоздаватьОбъектВПриемникеЕслиНеНайденПоПолямПоиска`,
  `ПродолжитьПоискПоПолямПоискаЕслиПоИдентификаторуНеНашли`, `ПравилоДляГруппыСправочника`,
  `ОбъектСДвижениями`, `Отключить`, `Захвачен`, `Копия`, `ВременнаяКопия`,
  `ВстроенныйОбработчикПослеЗагрузкиВсехДанных`,
  `ПриПереносеОбъектаПоСсылкеУстанавливатьТолькоGIUD`
- `Порядок` - Number; `Описание` - String
- Алгоритмы-обработчики (String): `АлгоритмПриОтправкеДанных`, `АлгоритмПриПолученииДанных`,
  `АлгоритмПередОтправкойДанных`, `АлгоритмПередПолучениемДанных`,
  `АлгоритмПриВыгрузкеОбъекта`, `АлгоритмПриЗагрузкеОбъекта`, `АлгоритмПередВыгрузкойОбъекта`,
  `АлгоритмПередЗагрузкойОбъекта`, `АлгоритмПослеВыгрузкиОбъекта`, `АлгоритмПослеЗагрузкиОбъекта`,
  `АлгоритмПослеЗагрузкиВсехДанных`, `АлгоритмПослеКонвертацииОбъекта`,
  `АлгоритмПриУдаленииОбъектаИБ`, `АлгоритмПоследовательностьПолейПоиска` и др.

Табличные части: `ИдентификацияПоПолямПоиска` (РеквизитыПоиска),
`НастройкаВариантовПоискаОбъектовXML` (ИмяНастройкиДляАлгоритма/ДляПользователя),
`Теги` (Тег - CatalogRef.Теги), `УдалитьПравилаКонвертацииСвойств` (устаревшая).

`АлгоритмПриОтправкеДанных`/`АлгоритмПриПолученииДанных` - тексты BSL обработчиков
конвертации (ИБ <-> XDTO). В генерируемом менеджере становятся процедурами.

## Справочник.ПравилаКонвертацииСвойств (ПКС, Владелец=ПКО, иерарх.)

Реквизиты (тип):
- `СвойствоКонфигурации`, `СвойствоКонфигурацииКорреспондента` - CatalogRef.Свойства
- `СвойствоКонфигурацииСтрокой`, `СвойствоФормата` - String
- `ПолноеИмяСвойстваКонфигурации`, `ПолноеИмяСвойстваФормата`,
  `ПолноеИмяСвойстваКонфигурацииКорреспондента` - String
- `ТипСвойстваКонфигурации`, `ТипСвойстваФормата`, `ТипСвойстваКонфигурацииКорреспондента` - String
- `ПравилоКонвертацииСвойства` - **String** (имя ПКО для ссылочного свойства, не ссылка)
- `Поиск` - Boolean (поле поиска; индексируется) - аналог прежнего «искать по свойству»
- `ИспользуетсяАлгоритмКонвертации` - Boolean (включает алгоритм)
- `АлгоритмПередВыгрузкойСвойства`, `АлгоритмПриВыгрузкеСвойства`,
  `АлгоритмПослеВыгрузкиСвойства`, `АлгоритмПередОбработкойВыгрузки`,
  `АлгоритмПослеОбработкиВыгрузки` - String (тексты BSL)
- `НеЗамещать`, `ПоискПоДатеНаРавенство`, `ПолучитьИзВходящихДанных`,
  `АвтоматическиПриводитьЗначениеКДлинеПриемника`, `ВыгружатьГруппуЧерезФайл`,
  `Отключить`, `Проверено` - Boolean
- `ИмяПараметраДляПередачи`, `Комментарий` - String; `Порядок` - Number
- `УсловиеПрименения` - CatalogRef.УсловияПримененияПравил

ПКС связывает свойство конфигурации (`СвойствоКонфигурации`) и свойство формата
(`СвойствоФормата`, строкой). Для нестандартного заполнения - алгоритм + флаг
`ИспользуетсяАлгоритмКонвертации`. ПКС создаётся как элемент справочника, Владелец=ПКО.

## Теневая модель метаданных

### Справочник.Конфигурации (топ-контейнер)
Реально только `УдалитьТегВырезки` (String, устаревший) + ТЧ
`УдалитьПодчиненныеКонфигурации` (устаревшая). Это просто контейнер; версии и
структура живут в Релизах, которые ссылаются на Конфигурацию.

### Справочник.Релизы (топ)
Версия загруженной структуры конфигурации.
- `КонфигурацияВладелец` - CatalogRef.Конфигурации (ShowError)
- `Версия`, `Имя`, `Синоним`, `Комментарий` - String
- `Приложение` - EnumRef.Приложения; `ИсточникДанных`, `ВариантЗагрузкиДвижений` - Number
- `ЕстьРасширения`, `ОбновлятьМетаданныеПоРасписанию` - Boolean
- `ДатаОбновления` - Date
- Git-поля (String): `АдресGitСервера`, `ИмяВеткиРепозитория`, `КаталогGitРепозитория`
- `КаталогЗагрузки`, `РегламентноеЗаданиеGUID` - String
- ТЧ `Расширения` (`ИмяКаталогаЗагрузки` - String)

Релизов 0 -> структура конфигурации не загружена, привязывать правила не к чему.

### Справочник.Объекты (Владелец=Релизы, иерарх.)
- `Имя`, `Синоним`, `Комментарий` - String
- `Тип` - **EnumRef.ТипыОбъектов** (не строка)
- `Иерархический`, `ОграничиватьКоличествоУровней`, `КонтрольУникальности`,
  `АвтоНумерация`, `Подчиненный` - Boolean
- `ВидИерархии`, `СерииКодов`, `Периодичность` - String; `КоличествоУровней` - Number
- `Идентификатор` - String (индекс)
- `ОбъектВышестоящегоРелиза` - CatalogRef.Объекты

### Справочник.Свойства (Владелец=Объекты, иерарх.)
- `Синоним`, `Комментарий`, `Использование`, `ТипыСтрокой`, `Подсказка` - String
- `Вид` - EnumRef.ВидыСвойств; `ВариантИндексирования` - EnumRef.ВариантыИндексирования
- `Индексирование`, `Авторегистрация` - Boolean
- Квалификаторы: `КвалификаторыЧисла_Длина`/`_Точность` - Number,
  `КвалификаторыЧисла_Неотрицательное` - Boolean, `КвалификаторыСтроки_Длина` - Number,
  `КвалификаторыСтроки_Фиксированная` - Boolean, `КвалификаторыДаты_Состав` - String
- `Идентификатор` - String (индекс); `СвойствоВышестоящегоРелиза` - CatalogRef.Свойства
- ТЧ `Типы` (`Тип` - CatalogRef.Объекты)

### Справочник.Значения (Владелец=Объекты, иерарх.)
Значения перечислений/предопределённые. `Синоним`, `Комментарий` - String;
`Предопределенное` - Boolean; `Идентификатор` - String. ТЧ `Типы` (Тип - CatalogRef.Объекты).

## Слой формата EnterpriseData

Это **цепочка справочников** (не табличные части ВерсииФормата!).

### Справочник.ВерсииФормата (топ, иерарх.)
- `ПространствоИмен` - String (namespace; НЕ «URIПространстваИмен»)
- `Расширение` - Boolean (признак: версия-расширение формата)
- `ДоступенЗапросНаВыгрузку` - Boolean
- **Табличных частей нет.** Состав формата - в подчинённых справочниках ниже.

Расширение формата = запись ВерсииФормата с `Расширение=Истина` и своим
`ПространствоИмен`.

### Справочник.ОбъектыФормата (Владелец=ВерсииФормата, иерарх.)
Наименование = имя объекта формата. `ЭтоГруппаОбщихСвойств` - Boolean;
`ТипОбщегоРеквизита` - EnumRef.ТипыОбщихРеквизитов; `ПереносОписанийВыполнен` - Boolean.

### Справочник.СвойстваФормата (Владелец=ОбъектыФормата, иерарх.)
- `Тип`, `ОбъектХранительСвойств` - String
- `Обязательное`, `СоставнойТип` - Boolean; `Порядок`, `НижняяГраница` - Number
- `ТипОбщегоСвойства` - EnumRef.ТипыОбщихРеквизитов
- ТЧ `Состав` (`ТипФормата`, `ИмяТипа` - String), `Фасеты` (`ВидФасета`, `ЗначениеФасета` - String)

### Справочник.ТипыФормата (Владелец=ВерсииФормата)
`БазовыйТип`, `ПространствоИмен`, `ПространствоИменБазовогоТипа` - String;
`СоставнойТип` - Boolean. ТЧ `Состав`, `Фасеты`.

### Справочник.ЗначенияФормата (Владелец=ОбъектыФормата, СвойстваФормата)
`Комментарий` - String, `Порядок` - Number.
