# Объектная модель КД 2.0

Все имена справочников и реквизитов получены выгрузкой `DumpConfigToFiles`
конфигурации `КонвертацияДанныхРедакция21` v2.1.8.2 и проверены на реальной
ИБ через `execute_query` MCP-toolkit (2 конвертации, 100 ПКО, 879 ПКС, 999 ПКЗ).

## Иерархия Владелец / Родитель

```
Конфигурации             (плоский справочник; описывает источник или приёмник)
  └─ Объекты             (Владелец=Конфигурации) - объекты метаданных
       └─ Свойства       (Владелец=Объекты) - реквизиты объекта
            └─ Значения  (Владелец=Свойства) - значения перечислений

Конвертации              (плоский; пара «Источник -> Приёмник»)
  ├─ ПКО                 (Владелец=Конвертации, может иметь Родитель=ПКО-группа)
  │   ├─ ПКС             (Владелец=ПКО)
  │   └─ ПКЗ             (Владелец=ПКО)
  ├─ ПВД                 (Владелец=Конвертации)
  ├─ ПОД                 (Владелец=Конвертации)
  ├─ Алгоритмы           (Владелец=Конвертации)
  ├─ Запросы             (Владелец=Конвертации)
  └─ Параметры           (Владелец=Конвертации)

Регистрации              (плоский)
  └─ ПРО                 (Владелец=Регистрации)
```

Все основные справочники - иерархические `HierarchyFoldersAndItems` с
`LevelCount=2`. Папки (`ЭтоГруппа=true`) используются для логической
группировки внутри одной Конвертации/Регистрации.

## Аббревиатуры

| Сокр. | Полное имя | Назначение |
|---|---|---|
| ПКО | Правило Конвертации Объекта | Конвертация объекта источника в объект приёмника |
| ПКС | Правило Конвертации Свойства | Конвертация одного реквизита |
| ПКГС | Правило Конвертации Группы Свойств | ПКС с `ЭтоГруппа = Истина` |
| ПКЗ | Правило Конвертации Значения | Конвертация значений перечислений |
| ПРО | Правило Регистрации Объекта | Когда регистрировать объект к обмену |
| ПВД | Правила Выгрузки Данных | Что и как выгружать |
| ПОД | Правила Очистки Данных | Что удалять при загрузке |

## Справочник.Конфигурации

Описание одной конфигурации-источника или конфигурации-приёмника. Хранит
имя, версию, и через подчиненные `Объекты` - теневую копию метаданных.

| Реквизит | Тип | Назначение |
|---|---|---|
| Наименование | string | Видимое имя (например `БухгалтерияПредприятияКОРП`) |
| Имя | string | Программное имя (обычно совпадает с Наименование) |
| Синоним | string | Русское описание |
| Комментарий | string | |
| Версия | string | Например `3.0.189.29` |
| ДатаОбновления | dateTime | |
| Приложение | EnumRef.Приложения | Например `Предприятие80` |

## Справочник.Объекты (Владелец=Конфигурации)

Объекты метаданных конкретной конфигурации.

| Реквизит | Тип | Назначение |
|---|---|---|
| Наименование, Имя | string | Имя объекта (`АвансовыйОтчет`, `Контрагенты`) |
| Синоним, Комментарий | string | |
| Тип | EnumRef.ТипыОбъектов | `НаборКонстант`, `Число`, `Строка`, `Справочник`, `Документ`, `РегистрСведений` и т.д. |
| Иерархический | boolean | |
| ВидИерархии | string | |
| ОграничиватьКоличествоУровней, КоличествоУровней | | |
| СерииКодов, КонтрольУникальности, АвтоНумерация | | Для справочников |
| Периодичность | string | Для регистров сведений |
| Подчиненный | boolean | |

Представление ссылок: `ДокументСсылка.АвансовыйОтчет`,
`СправочникСсылка.Контрагенты`.

## Справочник.Свойства (Владелец=Объекты)

Реквизиты объектов метаданных. Хранит тип, длину, ссылочность.

## Справочник.Значения (Владелец=Свойства)

Значения перечислений, предопределенные значения справочников.

### Группы свойств (ПКГС)

Реализуется через `ЭтоГруппа=true` на ПКС. Дочерние ПКС имеют `Родитель=ПКГС`.

### Типичное наименование

`<ИмяСвойства> --> <ИмяСвойства>`. Примеры из реальной ИБ:
- `Наименование --> Наименование`
- `Доп_НСИ_ГУИД --> Доп_НСИ_ГУИД`
- `Владелец --> Владелец`

## Паттерны заполнения (реальные данные из ИБ)

### ПКО «АвансовыйОтчетВДокументыВИБ» (БП КОРП -> УПП)

- Владелец = Конвертация `EXCH-0000001 (2025) - БП КОРП -> УПП`
- Родитель = ПКО-группа `Документы`
- Источник = Объекты `ДокументСсылка.АвансовыйОтчет` (из БП КОРП)
- Приемник = Объекты `СправочникСсылка.Доп_ДокументыВИБ` (из УПП)
- Порядок = 850
- `ИспользоватьБыстрыйПоискПриЗагрузке = true`, остальные boolean = false
- `ПриоритетОбъектовОбмена = ПриоритетПоступившегоПриОбменеОбъектаВыше`
- Все алгоритмы пустые

### ПКС «Наименование --> Наименование» (внутри ПКО `Валюты`)

- Владелец = ПКО `Валюты` (с trailing-пробелами в наименовании)
- Источник/Приемник = Свойства из соответствующих Объекты
- ТипИсточника/ТипПриемника = `Строка`
- ВидИсточника/ВидПриемника = `Свойство`
- Порядок = 100
- `Поиск = false`, обычное копирование Строка->Строка

### Конвертация «EXCH-0000001 (2025) - БП КОРП -> УПП»

- Код = `00000000-0000-0000-0000-000000000001    ` (UUID + trailing-пробелы)
- РежимСовместимости = `РежимСовместимостиСБСП20`
- Все 16 алгоритмов пустые
- `ВыгружатьПараметрыПоВерсии2_01 = true`

## Источники XML-метаданных

При необходимости получить полный список реквизитов или табчастей -
читать XML конфигурации:

- `C:\EDT\KD\src\Catalogs\<имя>.xml` (выгрузка `DumpConfigToFiles` от
  2026-05-20).

Файлы XML содержат блоки `<Attribute>` с реквизитами, `<TabularSection>` с
табчастями.
