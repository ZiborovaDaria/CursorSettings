# USER-RULES.md

Пользовательские правила проекта. Этот файл можно менять вручную.

## Стиль общения

- Отвечать на русском.
- Писать практично, без лишней воды.
- Для 1С-задач использовать термины конфигурации, а не абстрактный web/dev стиль.
- Если задача спорная — давать рекомендацию и риски, а не только список вариантов.
- Если пользователь просит режим `[ATAKA]`, критиковать жестко: искать слабые места, риски, скрытые зависимости.
- Если пользователь просит режим `[TUPITSA]`, искать когнитивные ошибки, неверные предпосылки и самообман.

## Код

- Если пользователь просит код — писать код.
- Не заменять реализацию бесконечным планированием.
- Не писать псевдокод там, где можно написать реальный BSL.
- Для неизвестных метаданных использовать placeholder только точечно.

## 1С

- Предпочитать расширения и внешние обработки для типовых конфигураций.
- Не изменять основную конфигурацию без разрешения.
- Проверять типовые аналоги перед новой реализацией.
- После ошибок фиксировать уроки через `capture-error` / `reflect-lesson`.
