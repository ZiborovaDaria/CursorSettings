# Грабли toolkit :6011 и КД 3.1 (проверено на живой ИБ)

Каждый пункт наблюдался фактически при работе с КД 3.1.6.14 через MCP-toolkit.

## Транспорт и кодировка

### Payload-файл, не -d с кириллицей

`curl -d '{"code":"...кириллица..."}'` из bash на Windows бьёт кириллицу.
Формировать JSON через `jq -Rs '{code: .}'` (fallback `python json.dumps`) в файл,
слать `curl --data-binary @файл`. Обёртки - `scripts/kd31_query.sh`, `kd31_exec.sh`.

### Ответ читать из файла через Read

Вывод curl в терминал портит кириллицу (видно как `Прав著...`) и визуально
обрезает строку (кажется, что вернулось ~10 элементов из 46). Сохранять ответ в
файл (`curl ... > out.json`) и читать инструментом Read - тогда UTF-8 корректен и
виден весь результат. Это критично: на искажённом stdout легко сделать ложный
вывод о структуре.

### Формат ответа

`execute_code` возвращает `{"success": true, "data": "{\"#\",\"value\":\"...\"}"}` -
полезное в `value`. Нормальный формат данной сборки toolkit.

## execute_code

### Серверный контекст, без `Возврат`

Результат - через переменную `Результат = ...`, не `Возврат`.

### `Справочники.X.Количество()` не работает

`Справочники.Конвертации.Количество()` падает «Метод объекта не обнаружен
(Количество)». Считать через запрос: `ВЫБРАТЬ КОЛИЧЕСТВО(*) КАК К ИЗ Справочник.X`.

### Правильная коллекция метаданных (ПКС - справочник)

`Метаданные.РегистрыСведений.ПравилаКонвертацииСвойств` падает «Поле объекта не
обнаружено» - ПКС в КД 3.1 это **справочник**:
`Метаданные.Справочники.ПравилаКонвертацииСвойств`. При «Поле объекта не
обнаружено (Имя)» - проверить, в той ли коллекции (Справочники/РегистрыСведений/
Обработки) ищется объект, и точное имя через `Метаданные.Справочники` (список).

### «Неопознанный оператор» - упрощать код

Многострочный BSL (циклы, `Если`) в целом исполняется. Но `ВРег("...")` с
вложенными кавычками внутри `Если` давал «Неопознанный оператор». При такой ошибке
- убрать подозрительную функцию, вынести строку в переменную, сократить код.

### Длинные перечисления - в файл

Сбор длинных строк (списки объектов/реквизитов) - сохранять ответ в файл и читать
Read; на stdout строка обрезается визуально.

## Поведение КД 3.1

### Релизы = 0 - структура конфигурации не загружена

`ВЫБРАТЬ КОЛИЧЕСТВО(*) ИЗ Справочник.Релизы` = 0 означает, что структура
конфигурации не загружена (обработка `ЗагрузкаСтруктурыКонфигурацииИзФайловXML`).
Без Релиза справочники `Объекты`/`Свойства` пусты, ПКО привязать не к чему.
Поля Конвертации `Конфигурация`/`КонфигурацияКорреспондент` ссылаются на Релизы.

### ВерсияФорматаМенеджера - короткий реквизит

Реквизит `ВерсияФорматаМенеджера` короткий (по наблюдениям - 2 символа): строка
вида `"1.10"` молча обрезается до `"1."`. Для двузначного минора хранить иначе
(полная версия - в URI пространства имён версии формата), либо проверять записанное
значение после `Записать()`.

### Короткие имена перечислений

Имена значений перечислений КД 3.1 короткие. Выверено по EDT:
- `Перечисления.ТипыКонвертаций` - `XML`, `XDTO` (не `ФорматXDTO`).
- `Перечисления.ВариантыМоделейОбмена` - `ПакетнаяМодель`, `ОбъектнаяМодель`,
  `КомбинированнаяМодель` (тип реквизита `Конвертации.ИнтерфейсКонвертации`; перечисления
  `ИнтерфейсыКонвертации` НЕТ).
- `Перечисления.ВариантыИдентификацииОбъектов` - `ПоУникальномуИдентификатору`,
  `ПоПолямПоиска`, `СначалаПоУникальномуИдентификаторуПотомПоПолямПоиска` (перечисления
  `ВариантыИдентификацииОбъектовПКО` НЕТ).
- `Перечисления.СостоянияПравил` - `Черновик`, `ВРаботе`, `НаТестировании`, `Готово`,
  `Архив` (тип реквизита `Конвертации.Статус`).
Перед использованием значения перечисления - свериться со списком значений.

### Имена обработок - сверять с фактическими

В этой сборке генерация модуля - обработка `ВыгрузкаМодуля`, загрузка структуры -
`ЗагрузкаСтруктурыКонфигурацииИзФайловXML`, редактор правил - `НастройкаПравилОбмена`.
Имён `ВыгрузкаПравил`/`ПодключениеКорреспондента`/`ПомощникСозданияФорматаXDTO` нет.
Перед вызовом обработки - проверить имя через список `Метаданные.Обработки`.

### Загрузчик УДАЛЯЕТ поданные файлы (XSD и .mdo)

И `ЗагрузкаСтруктурыКонфигурацииИзФайловXML.ВыполнитьЗагрузкуМетаданных`, и
`ЗагрузкаСтруктурыФормата.ВыполнитьЗагрузкуФормata` в КОНЦЕ работы делают
`УдалитьФайлы(...)` по каждому поданному файлу (`Хранение` для .mdo,
`ФайлыФабрики` для XSD). ПОДТВЕРЖДЕНО: поданные напрямую XSD базового формата были
СНЕСЕНЫ после загрузки. ВСЕГДА подавать файлы (XSD расширения, .mdo) из ВРЕМЕННОЙ
КОПИИ (robocopy/cp), не из постоянного места и не из EDT-оригиналов.

### Тяжёлая загрузка занимает сеанс toolkit (HTTP 000 у параллельных запросов)

Загрузка структуры (17к объектов, ~30 мин) держит сеанс 1С занятым - MCP_Toolkit
однопоточный, параллельные запросы к :6011 возвращают HTTP 000 (не отвечает) до
завершения. Это НЕ падение. Опрашивать готовность циклом `curl get_metadata` до
HTTP 200, только потом проверять результат. Сам факт HTTP Gateway Timeout у
запускающего вызова - норма (см. выше), 1С продолжает в своём сеансе.

## Запись / удаление

Если в форме toolkit включена защита (`Записать`/`Удалить`/
`УстановитьПривилегированныйРежим`) - для правки её снять. `Удалить()` в обычных
формах может зависнуть на модальном подтверждении - использовать
`УстановитьПометкуУдаления(Истина)`, физическое удаление через UI КД. Работать на
тестовой копии ИБ КД, не на боевой.
