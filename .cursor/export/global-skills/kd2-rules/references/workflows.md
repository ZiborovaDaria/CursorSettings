# Сценарии работы с КД 2.0

End-to-end сценарии с готовыми curl-командами к MCP-toolkit (порт `:7003`).
Замени порт на свой, если toolkit запущен по другому адресу.

## W0. Проверка готовности

```sh
# Toolkit живой?
curl http://localhost:7003/health

# КД отвечает?
curl -sS -X POST "http://localhost:7003/api/execute_query" \
  -H "Content-Type: application/json" \
  -d '{"query":"ВЫБРАТЬ \"OK\" КАК Статус"}'

# Сколько конвертаций уже есть?
curl -sS -X POST "http://localhost:7003/api/execute_query" \
  -H "Content-Type: application/json" \
  -d '{"query":"ВЫБРАТЬ Наименование ИЗ Справочник.Конвертации ГДЕ НЕ ЭтоГруппа"}'

# Список конфигураций?
curl -sS -X POST "http://localhost:7003/api/execute_query" \
  -H "Content-Type: application/json" \
  -d '{"query":"ВЫБРАТЬ Наименование, Версия ИЗ Справочник.Конфигурации"}'
```

## W1. Загрузить метаданные конфигурации источника в КД

Предусловие: у тебя есть XML с метаданными конфигурации (получен через
`Обработки.ВыгрузкаМетаданных` в самой ИБ источника).

```sh
curl -sS -X POST "http://localhost:7003/api/execute_code" \
  -H "Content-Type: application/json" \
  -d '{"code":"Обр = Обработки.ЗагрузкаСтруктурыМетаданных.Создать(); Обр.ИмяФайлаЗагрузки = \"C:\\\\Input\\\\metadata_source.xml\"; Обр.мЗагрузитьНовую = Истина; Обр.мКомментарий = \"Загрузка метаданных БП КОРП\"; Рез = Обр.ВыполнитьЗагрузку(); Результат = Новый Структура(\"ok,result\", Рез <> Ложь, Строка(Рез));"}'
```

После загрузки:

```sh
# Проверить, что Конфигурация появилась
curl -sS -X POST "http://localhost:7003/api/execute_query" \
  -H "Content-Type: application/json" \
  -d '{"query":"ВЫБРАТЬ Наименование, Имя, Версия ИЗ Справочник.Конфигурации УПОРЯДОЧИТЬ ПО Наименование"}'

# Сколько объектов под этой Конфигурации?
curl -sS -X POST "http://localhost:7003/api/execute_query" \
  -H "Content-Type: application/json" \
  -d '{"query":"ВЫБРАТЬ Конфигурации.Наименование КАК Конфигурация, КОЛИЧЕСТВО(Объекты.Ссылка) КАК Кол ИЗ Справочник.Конфигурации КАК Конфигурации ЛЕВОЕ СОЕДИНЕНИЕ Справочник.Объекты КАК Объекты ПО Объекты.Владелец = Конфигурации.Ссылка СГРУППИРОВАТЬ ПО Конфигурации.Наименование"}'
```

## W2. Создать конвертацию

После того как обе Конфигурации (Источник, Приёмник) загружены.

```sh
curl -sS -X POST "http://localhost:7003/api/execute_code" \
  -H "Content-Type: application/json" \
  -d @- <<'JSON'
{
  "code": "
Источник = Справочники.Конфигурации.НайтиПоНаименованию(\"БухгалтерияПредприятияКОРП\");
Приемник = Справочники.Конфигурации.НайтиПоНаименованию(\"УправлениеПроизводственнымПредприятием\");
Если Источник.Пустая() ИЛИ Приемник.Пустая() Тогда
    ВызватьИсключение \"Источник или Приёмник не найдены\";
КонецЕсли;

Конв = Справочники.Конвертации.СоздатьЭлемент();
Конв.Наименование = \"Тест: БП КОРП -> УПП\";
Конв.Источник = Источник;
Конв.Приемник = Приемник;
Конв.ИмяИсточника = Источник.Имя;
Конв.ИмяПриемника = Приемник.Имя;
Конв.Код = Строка(Новый УникальныйИдентификатор());
Конв.РежимСовместимости = Перечисления.РежимСовместимости.РежимСовместимостиСБСП20;
Конв.ВыгружатьПараметрыПоВерсии2_01 = Истина;
Конв.Записать();

Результат = Новый Структура(\"ссылка,код\", XMLСтрока(Конв.Ссылка), СокрЛП(Конв.Код));
"
}
JSON
```

## W3. Создать ПКО

Предполагаем, что Конвертация и оба объекта (Источник, Приёмник) уже
существуют в КД.

```bsl
Конв = Справочники.Конвертации.НайтиПоНаименованию("Тест: БП КОРП -> УПП");
ОбъИст = Справочники.Объекты.НайтиПоНаименованию("Контрагенты", Истина, Конв.Источник);
ОбъПр  = Справочники.Объекты.НайтиПоНаименованию("Контрагенты", Истина, Конв.Приемник);
Если ОбъИст.Пустая() ИЛИ ОбъПр.Пустая() Тогда
    ВызватьИсключение "Объекты не найдены";
КонецЕсли;

ПКО = Справочники.ПравилаКонвертацииОбъектов.СоздатьЭлемент();
ПКО.Владелец      = Конв;
ПКО.Наименование  = "Контрагенты";
ПКО.Источник      = ОбъИст;
ПКО.Приемник      = ОбъПр;
ПКО.ТипИсточника  = "СправочникСсылка.Контрагенты";
ПКО.ТипПриемника  = "СправочникСсылка.Контрагенты";
ПКО.Порядок       = 100;
ПКО.Код           = Строка(Новый УникальныйИдентификатор());
ПКО.НеЗамещать    = Истина;
ПКО.НеВыгружатьОбъектыСвойствПоСсылкам = Истина;
ПКО.НеСоздаватьОбъектВПриемникеЕслиНеНайденПоПолямПоиска = Истина;
ПКО.Записать();

Результат = XMLСтрока(ПКО.Ссылка);
```

## W4. Создать ПКС внутри ПКО

```bsl
ПКО = Справочники.ПравилаКонвертацииОбъектов.НайтиПоНаименованию("Контрагенты", Истина, Конв);
СвИст = Справочники.Свойства.НайтиПоНаименованию("Наименование", Истина, ПКО.Источник);
СвПр  = Справочники.Свойства.НайтиПоНаименованию("Наименование", Истина, ПКО.Приемник);

ПКС = Справочники.ПравилаКонвертацииСвойств.СоздатьЭлемент();
ПКС.Владелец     = ПКО;
ПКС.Источник     = СвИст;
ПКС.Приемник     = СвПр;
ПКС.ИмяИсточника = "Наименование";
ПКС.ИмяПриемника = "Наименование";
ПКС.ТипИсточника = "Строка";
ПКС.ТипПриемника = "Строка";
ПКС.ВидИсточника = "Свойство";
ПКС.ВидПриемника = "Свойство";
ПКС.Наименование = "Наименование --> Наименование";
ПКС.Порядок      = 100;
ПКС.Поиск        = Ложь;
ПКС.Код          = Строка(Новый УникальныйИдентификатор());
ПКС.Записать();
```

### ПКС с ссылочным свойством (через `ПравилоКонвертации`)

Когда свойство имеет ссылочный тип (например `Владелец` у договора - ссылка
на `Контрагенты`), указать `ПравилоКонвертации` - ссылку на ПКО для
конвертации значения ссылки.

```bsl
ПКОВладелец = Справочники.ПравилаКонвертацииОбъектов.НайтиПоНаименованию("Контрагенты", Истина, Конв);

ПКС.Источник     = СвВладелец_Ист;
ПКС.Приемник     = СвВладелец_Пр;
ПКС.ИмяИсточника = "Владелец";
ПКС.ИмяПриемника = "Владелец";
ПКС.ТипИсточника = "СправочникСсылка.Контрагенты";
ПКС.ТипПриемника = "СправочникСсылка.Контрагенты";
ПКС.ВидИсточника = "Свойство";
ПКС.ВидПриемника = "Свойство";
ПКС.ПравилоКонвертации   = ПКОВладелец;
ПКС.ИмяПравилаКонвертации = "Контрагенты";
ПКС.Записать();
```

## W5. Создать ПКЗ (правило конвертации значения перечисления)

```bsl
ПКО = Справочники.ПравилаКонвертацииОбъектов.НайтиПоНаименованию("СтатусыЗаказа", Истина, Конв);
ЗнИст = Справочники.Значения.НайтиПоНаименованию("ВРаботе", Истина, СвойствоИсточника);
ЗнПр  = Справочники.Значения.НайтиПоНаименованию("InProgress", Истина, СвойствоПриёмника);

ПКЗ = Справочники.ПравилаКонвертацииЗначений.СоздатьЭлемент();
ПКЗ.Владелец         = ПКО;
ПКЗ.Источник         = ЗнИст;
ПКЗ.Приемник         = ЗнПр;
ПКЗ.ЗначениеИсточник = "ВРаботе";
ПКЗ.ЗначениеПриемник = "InProgress";
ПКЗ.Порядок          = 100;
ПКЗ.Записать();
```

## W6. Выгрузить итоговый XML

```bsl
Конв = Справочники.Конвертации.НайтиПоНаименованию("Тест: БП КОРП -> УПП");

Обр = Обработки.ВыгрузкаКонвертации.Создать();
Обр.КонвертацияДляВыгрузки         = Конв;
Обр.ИмяФайлаПравил                 = "C:\Output\test_rules.xml";
Обр.СжиматьФайлПравил              = Ложь;
Обр.ВыгружатьПравилаКорреспондента = Ложь;
Обр.ВыгружатьПравилаРегистрации    = Ложь;
Обр.ВыполнитьВыгрузкуПравил();

Файл = Новый Файл(Обр.ИмяФайлаПравил);
Результат = Новый Структура("ok,path,size", Истина, Обр.ИмяФайлаПравил, Файл.Размер());
```

После - прочитать XML и сверить с эталоном при необходимости.

## W7. Импорт существующих правил из XML (для рефлексии)

```bsl
Обр = Обработки.ЗагрузкаКонвертации.Создать();
Обр.ЗагрузитьПравилаОбмена("C:\Input\existing_rules.xml", "XMLФайл");
// При необходимости вызвать Обр.Сохранить(...) для финального сохранения
```

После импорта - правила появятся в справочниках КД, можно изучать структуру.

## W8. Автонастройка правил (быстрый стартер)

После того как обе Конфигурации в КД:

```bsl
Обр = Обработки.АвтонастройкаПравилКонвертацииОбъектов.Создать();
// Заполнить реквизиты (Конвертация-цель, режимы)
// Запустить (имя метода - проверить в src/DataProcessors/АвтонастройкаПравилКонвертацииОбъектов/Ext/ObjectModule.bsl)
```

Аналогично - `АвтонастройкаПравилКонвертацииСвойств` и
`АвтонастройкаПравилКонвертацииЗначений`. Полезно сразу после `W1` создать
каркас правил, а потом дорабатывать руками.

## W9. Полный цикл (от пустой КД до XML)

```
1. W0 - проверить готовность toolkit-а и пустую/тестовую КД
2. W1 - загрузить XML метаданных Источника (получить через ВыгрузкаМетаданных в ИБ источника)
3. W1 - загрузить XML метаданных Приёмника
4. W2 - создать Конвертацию (Источник, Приёмник)
5. W8 - запустить АвтонастройкаПравилКонвертацииОбъектов (создать каркас ПКО)
6. W8 - АвтонастройкаПравилКонвертацииСвойств (заполнить ПКС внутри ПКО)
7. W8 - АвтонастройкаПравилКонвертацииЗначений (для перечислений)
8. (опционально) W3-W5 - доработать правила руками
9. W6 - выгрузить итоговый XML
10. (опционально) ВыгрузкаОбработчиков - если есть алгоритмы событий
```

## Откат изменений

КД не имеет «отмены последнего действия». Перед массовыми изменениями:
- Сделать резервную копию ИБ (для файловой - просто копия каталога).
- Или выгрузить текущее состояние через `W6` как backup.
- Или работать на тестовой копии ИБ, а не на production.

Если нужно удалить созданную тестовую конвертацию:

```bsl
Ссылка = Справочники.Конвертации.НайтиПоНаименованию("Тест: БП КОРП -> УПП");
Объ = Ссылка.ПолучитьОбъект();
Объ.УстановитьПометкуУдаления(Истина);
// Помеченные записи физически удаляются через UI КД:
// «Все функции» -> «Стандартные» -> «Удаление помеченных объектов»
```

**Важно:** прямое `Объ.Удалить()` через MCP-toolkit зависает (модальный
диалог в обычных формах). Использовать `УстановитьПометкуУдаления`.

Связанные ПКО/ПКС/ПКЗ нужно помечать отдельно - они подчинены Конвертации
через `Владелец`, но автоматическое каскадное удаление не происходит:

```bsl
// Пометить все ПКО конвертации
Конв = Справочники.Конвертации.НайтиПоНаименованию("Тест: БП КОРП -> УПП");
Запрос = Новый Запрос("ВЫБРАТЬ Ссылка ИЗ Справочник.ПравилаКонвертацииОбъектов ГДЕ Владелец = &К");
Запрос.УстановитьПараметр("К", Конв);
Выб = Запрос.Выполнить().Выбрать();
Пока Выб.Следующий() Цикл
    Об = Выб.Ссылка.ПолучитьОбъект();
    Об.УстановитьПометкуУдаления(Истина);
КонецЦикла;
```
